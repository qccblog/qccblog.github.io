    grunt配置代理解决跨域

    qiucong 2018/1/10 grunt生产力

    在grunt中使用grunt-connect-proxy中可以简单的设置代理来解决前端请求后台API的时候发生跨域的错误。

    # 安装

    在使用fineui已有的Gruntfile配置的基础上，还需要安装两个包，一个用来做代理，一个用来访问本地文件。

    npm i -grunt-connect-proxy serve-static  --save-dev

    # 配置

    主要是配置Gruntfile.js文件。

    # 引入包

    var serveStatic = require(``'serve-static'``);

    # 修改connect

    connect: {
  options: {
    port: 9000,
    open: true,
    livereload: 35739,
    hostname: "localhost"
  },

  /* 设置代理 */
  proxies: [{
    context: "/api/v10",  // 设置要监听的url包含的字段
    host: "192.168.0.1",         // 后台服务器的ip
    port: 8080        // 后台服务器的端口
  }],

  //把原本的server改掉了
  livereload: {
    options: {
      port: 9002,
      base: "./",
      middleware: function (connect, options) {
        if (!Array.isArray(options.base)) {
          options.base = [options.base];
        }

        // Setup the proxy
        var middlewares = [require("grunt-connect-proxy/lib/utils").proxyRequest];

        // Serve static files.
        options.base.forEach(function (base) {
          middlewares.push(serveStatic(base));
        });

        // Make directory browse-able.
        var directory = options.directory || options.base[options.base.length - 1];
        middlewares.push(connect().use(directory, serveStatic(directory)));
        return middlewares;
      }
    }
  }
}

    # 注册任务

    grunt.loadNpmTasks("grunt-connect-proxy");

grunt.registerTask("server", function (target) {
  grunt.task.run([
    "less",
    "concat",
    "configureProxies:server", //一定要这个
    "connect:livereload",
    "watch"
  ]);
});

    # 使用

    在访问http://localhost:9002/api/v10/ (opens new window)形式的url时候，就会自动代理成设置的后台服务器的url了。这样就不会产生跨域的问题。
