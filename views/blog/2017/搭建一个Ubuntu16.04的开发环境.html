    搭建一个Ubuntu16.04的开发环境

    qiucong 2017/6/10 安装配置

    使用以下安装过程，可以搭建一个基本可用的开发环境，在自己需求的基础上选择需要安装的项目或者安装新的项目。

    1. 安装openssh，方便使用putty之类工具连接
      sudo apt-get install openssh-server

    2. 安装vim，自带的vi有方向键不能用等问题
      sudo apt-get install vim

    3. 设置root用户密码，默认是没有root用户密码的
      sudo passwd root

    4. 修改某个用户使用sudo不用密码，默认每次登录的第一次使用sudo都是需要输入密码

      #完全进入root用户环境，然后修改/etc/sudoers文件
su - root
#修改sudoers文件权限，默认不能写
cd /etc
chmod 777 sudoers
#修改文件内容，使用以下把用户加进去
#用户名     ALL=(ALL)NOPASSWD: ALL
vi sudoers
#修改会sudoers文件权限，不然sudo不能使用
chmod 440 sudoers
#使用Ctrl + d退出root，这时候直接使用sudo就行了，不需要再输入密码

    5. 安装samba用于共享文件夹，在虚拟机中开发的时候，共享文件夹就能在原系统中直接使用编辑器打开了。
      sudo apt-get install samba修改/etc/samba/smb.conf配置Samba，在smb.conf文件中底部添加如下内容：

      [share]
comment=this is Linux share directory
path=/home/share
browseable = yes
public=no
writable=yes

      设置smb用户密码，注意该用户名需要是系统中已经存在的用户。
      sudo smbpasswd -a \[username\] 在Windows的文件夹地址栏输入\\内网ip出现共享文件夹就会提示输入用户密码，然后就可以共享文件夹了。

    6. 使用静态ip，如果使用的虚拟机是VMware，最好在虚拟机的设置里面把网络模式改成桥接模式
      修改/etc/network/interfaces文件，增加

      auto eth0
iface eth0 inet static
address 10.1.0.128
netmask 255.255.255.0
gateway 10.1.0.1
dns-nameserver 8.8.8.8

    7. 配置php开发环境

      #安装php和Apache服务器
sudo apt-get install php
sudo apt-get install apache2
sudo apt-get install libapache2-mod-php7.0

      修改Apache2的使用的mods
      在/etc/apache2/mods-enabled/中使用sudo cp -s ../mods-available/选项 ./软链接过来
      根据php开发需要的功能进行选择性安装
      sudo apt-get install php-cli php-common php-gd php-mysql php-curl php-xml php-memcache php-intl php-mongodb php-memcached phpmyadmin 问题：
      1.PHP Fatal error: Undefined class constant ‘MYSQL_ATTR_INIT_COMMAND’
      如果使用PDO接口来访问MySQL数据库，但是PDO在php.ini配置默认是关闭的，开启php.ini配置中php_pdo_mysql.dll即可（去掉前面的；字符）

    8. 安装node和npm，默认版本都比较老
      sudo apt-get install nodejs-legacy npm

    9. 安装svn
      sudo apt install subversion

    10. 安装git
      sudo apt install git

    11. 配置多域名Apache2
      修改/etc/apache2/apache2.conf文件，使用shift+g到文件最底部，将
      IncludeOptional sites-enabled/*.conf的.conf删除以便Apache2能够读取sites-enabled下任意文件名的域名配置
      在/etc/apache2/sites-available里面新建文件配置新的域名，一般以域名为文件名以方便识别，一种格式如下：

      <Directory "/var/www/site">
Options -Indexes
AllowOverride All

Require all granted
</Directory>

<VirtualHost _default_:80>
DocumentRoot /var/www/site
ServerName www.test.com
ErrorLog /var/www/site/log/error.log
</VirtualHost>

    12. 安装mongodb
      sudo apt-get install mongodb 配置mongodb可以远程访问，编辑/etc/mongodb.conf文件
      把bind 127.0.0.1改成 bind 0.0.0.0

      ##给数据库添加用户

#进入mongo数据库
mongo
#选择数据库
use db_name
#添加用户名密码
db.addUser('user', 'password')

##添加整个mongo的管理员账户
use admin
db.addUser('user', 'password')

##备份数据库
mongodump --help
#还原数据库
mongorestore -d db_name --directoryperdb  文件夹 --drop
#导出数据库
mongoexport -d db_name -c collection_name --csv -q  -f 字段1,字段2 -o 文件名

    15. 永久修改主机名
      修改主机名是个不错的选择，修改/etc/hostname文件，把文件内容改成想要修改成的主机名，然后最好修改一下/etc/hosts文件加上127.0.1.1 name，这样就不会在你使用的时候老是告诉你说主机没找到什么的
