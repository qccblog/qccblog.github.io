    z-index学习

    qiucong 2017/12/24 css基础梳理

    以前一直用着z-index，也就是知道可以设置数值大的可以放在数值小的上面，大概就是知其然不知其所以然的样子。趁着这次使用，梳理了一下对z-index的理解。

    # 什么是z-index

    z-index用来控制元素重叠时堆叠顺序。

    z-index 属性指定了一个元素及其子元素的 z-order。 当元素之间重叠的时候，z-order 决定哪一个元素覆盖在其余元素的上方显示。 通常来说 z-index 较大的元素会覆盖较小的一个。

    对于一个已经定位的元素（即position属性值是非static的元素），z-index 属性指定：

    1. 元素在当前堆叠上下文中的堆叠层级。
    2. 元素是否创建一个新的本地堆叠上下文。

    z-index的默认值为auto，此时元素不会建立一个新的本地堆叠上下文。当前堆叠上下文中新生成的元素和父元素堆叠层级相同。

    整型数字是生成的元素在当前堆叠上下文中的堆叠层级。元素同时会创建一个堆叠层级为0的本地堆叠上下文。这意味着子元素的 z-indexes 不与元素外的其余元素的 z-indexes 进行对比。

    具体定义可以参考这里 (opens new window)

    # 重点

    我们需要重点关注的就是堆叠上下文这个东西。每个元素仅属于一个堆叠上下文，元素的z-index描述元素在相同堆叠上下文中“z轴”的呈现顺序。

    我觉得z-index的作用就是改变默认的排序方式，最主要的一点就是要知道当前元素默认值为auto的时候是没有创建堆叠上下文的，当前元素的堆叠上下文要一直往上找，直到找到有上下文的元素为止（最后就是html根元素）。兄弟堆叠上下文是独立的，只会影响到后代元素，后代的堆叠上下文不会影响到更高的元素。

    总的来说就是z-index就是在同一个层级之间比较，父层的z-index决定了其所有子元素在与父元素兄弟元素及其子元素比较时候的层级。（一个不太恰当的比喻，在有些制度的社会，你生来是什么身份其实早就在你父辈就已经决定好了，能做的就是在同一个层次的人中比较优劣）。

    需要注意的地方就是默认值为auto的情况，因为他自身没有堆叠上下文，需要一直往上找，所以最后就有可能是某个层级很深的子元素和外层元素进行比较了。这种情况比较容易出现在做一个弹出层的情况下。

      width: 200px;

      top: 0;
      left: 50%;
      margin-top: 100px;
      margin-left: -100px;
      background: red;
      z-index: 1;
    }

    .context11 {
      width: 300px;
      height: 150px;
      background: rebeccapurple;
    }

    .context12 {
      position: relative;
      width: 300px;
      height: 300px;
      z-index: 2;
      background: blue;
    }
  </style>

</head>

<body>

    <div id="box1">

      <div class="context11">context11</div>

      <div class="context12">context12</div>

    </div>

  </div>

  <div id="popup">

    我们希望的情况是弹出层会覆盖在wrap节点之上，但是实际上却不是这样的，context12元素就把popup给覆盖了，看了一下css发现是context12的z-index比popup的要大，所以就覆盖了，把popup的改大一点就行了呗。的确是可以的，但是我们开始说过，z-index其实是同一层次的进行比较，wrap里面的东西应该注定要比popup低才对，就算wrap里面再大也不应该出现在popup之上。这种情况下，我们可以给wrap设置一个z-index就可以解决问题了。

    出现这种情况的原因，就是context12的父元素的z-index都是auto，没有创建堆叠上下文，所以context12的父堆叠上下文就是根元素，同理，popup的父堆叠上下文也是根元素，这样就成了同级元素进行比较了，当然是谁大谁上了。但是，如果给wrap加一个z-index属性，context12的父堆叠上下文就是wrap了，wrap的子元素一定在层级比wrap高的元素之下了。
