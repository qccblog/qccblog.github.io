    # 使用crypto生成

    生成指定length长度的随机字符串，具体使用看文档 (opens new window) crypto.randomBytes(length).toString('hex')

    总感觉这样重复的概率可能会比较大，所有可以考虑在某个位置加上当前时间戳会好一点。 生成指定length长度的随机字符串，末尾是16进制的时间戳 crypto.randomBytes(length - 4).toString('hex') + (Date.parse(new Date())/1000).toString(16)

    # 使用Hashids生成字符串

    假定需要很多的链接来指向不同的内容，这时候生成一定长度的字符串就很有必要了；或者你不想在地址栏中暴露数据库中直观的Id之类的，可以使用Hashids生成一个字符串。Hashids主要是把数字(数字数组)转换成不可预测且唯一的短字符串，常见的语言基本上都支持。Hashids官网 (opens new window)

    # 安装
npm install --save hashids

    # 使用uuid

    在nodejs或者浏览器中生成符合标准的UUIDs。使用也非常简单，主要就是长度比较长，可以和Hashids根据不同场景互补使用。

    var uuid = require('uuid');
// 生成一个基于时间的v1版UUID
console.log(uuid.v1()); // -> '69e31ec0-66b8-13e7-b6de-1f71e6a5a9d9'

// 生成一个随机的v4版UUID
console.log(uuid.v4()); // -> '5cac2836-36fb-4aad-bda2-5915e98bcdb0'

    # 自己动手实现

    可以使用Math.random()来创建随机字符串，有很多种方法。一般都是根据Math.random()生成的随机数，然后处理该随机数（放大，取整），最后从给定的字符串中取出一个或多个，重复指定次数拼接成最后的字符串。 
    一个简单的示例，还有很多方法可以使其性能更优，实现更简便。

    function randomChar(len) {
  var str = "0123456789qwertyuioplkjhgfdsazxcvbnmQWERTYUIOPLKJHGFDSAZXCVBNM";
  var tmp = "";
  for(var i = 0; i < len; i++)  {
    tmp += str.charAt(Math.floor(Math.random() * str.length));
  }
  return tmp;
}
console.log(randomChar(10)); //输出长度为10的字符串

    在网上看到过一种很意思的写法Math.random().toString(36).substr(2);。就是把Math.random()生成的0-1的随机数转换成字符串类型，toString可以接受2-36范围的参数，可以理解为转换成相应的进制，36进制的话就相当于是包含0-9a-z的字符串集合。这样就可以生成0.xxxxxxx这种形式的字符串了，从第2位开始截取就能获得一定长度的字符串。 需要注意的是，toString函数最好不要传入奇数，不然对小数使用toString方法是一件很恐怖的事情。还有就是这个最后得出来的长度是不确定的(总长度可能12位或13位或14位…)。所以这个方法实际使用需要注意。
