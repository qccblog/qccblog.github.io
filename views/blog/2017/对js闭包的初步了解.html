    对js闭包的初步了解

    qiucong 2017/5/13 js基础梳理

    # 一、变量的作用域

    在js中变量分为全局变量和局部变量。函数内部可以访问全局变量，函数外部不能访问函数内部的局部变量。函数内部需要时用var来声明局部变量，不然就是声明了全局变量。在es6中，可以使用let和const来声明变量。

    # 二、闭包的形式

    1. 当return的是内部function时，就是一个闭包。内部function会保存引用外部function的变量直到内部function结束。

    # 三、闭包的作用

    1. 访问局部变量
    2. 保存变量的引用，让变量的值保存在内存当中
    function a () {
    var i = 1;
    function b () {
        return i++; 
    }
    return b;
}

//这三个效果是相同的
var aa = a();
var aa = new a;
var aa = new a();       //感觉如果使用new更好理解一些，可以把闭包理解为对象。

var bb = a();
console.log(typeof aa)  //function
console.log(aa());      //1
console.log(aa());      //2
console.log(bb());      //1

    个人理解：var aa = a();相当于是右侧执行了a函数，这样就把a函数return的值赋给了变量aa，这个时候aa就相当于是a函数里面的b函数，然后通过执行aa函数就是一个执行b函数的过程，这样就访问到了a函数内部的变量i，与此同时，变量i相当于一直在被aa函数使用，所以不会被js给自动回收。由于每个闭包都是var bb = a();相当于创建了一个新a()对象

    # 四、闭包的实例

    function foo(x) {
    var tmp = 3;
    return function (y) {
        alert(x + y + (++tmp));
    }
}
var bar = foo(2); // bar 现在是一个闭包
bar(10);


var db = (function() {
    // 创建一个隐藏的object, 这个object持有一些数据
    // 从外部是不能访问这个object的
    var data = {};
    // 创建一个函数, 这个函数提供一些访问data的数据的方法
    return function(key, val) {
        if (val === undefined) { return data[key] } // get
        else { return data[key] = val } // set
    }
    // 我们可以调用这个匿名方法
    // 返回这个内部函数，它是一个闭包
})();

db('x'); // 返回 undefined
db('x', 1); // 设置data['x']为1
db('x'); // 返回 1
// 我们不可能访问data这个object本身
// 但是我们可以设置它的成员


function closureExample(objID, text, timedelay) {
    setTimeout(function() {
        document.getElementById(objID).innerHTML = text;
    }, timedelay);
}
closureExample(‘myDiv’, ‘Closure is created’, 500);

    # 五、闭包实例分析

    我们希望定义一个函数，最后能够输出0~9

    function a(){
    var arr = new Array();
    for (var i=0; i < 10; i++){
        arr[i] = function(){
            return i;
        };
    }
    return arr;
}
var fn = a();
for (var i=0; i < fn.length; i++){
    console.log(fn[i]());
}

    这个例子最后输出的10个10，并不是我们预期的那样输出0~9。原因就是在函数a当中，给arr[i]赋值的函数仅仅是定义了，但是并没有被执行，这样就相当于最后返回arr[0] = function(){return i;}这种形式的，只有arr的键在增长，右边的函数并没有变。for里面的i最后自加到10，并且由于右边的匿名函数还在使用变量i，所以i不会被回收，这样i的值就为10。把函数a的返回值赋给fn之后，fn[0]就是function(){return i;}这个函数，这样输出的就都是最后的i值10。(函数声明是先被执行的)

    那么这个例子就不是闭包了吗？实际上还是一个常见的返回函数这种形式的闭包，只是函数没有被执行对这个例子产生了影响。改写一下这个函数可以产生不同的结果。

    function a(){
    var arr = new Array();
    for (var i=0; i < 10; i++){
        arr[i] = function(){
            return i++;     //只是改变了一下这里
        };
    }
    return arr;
}
var fn = a();
for (var i=0; i < fn.length; i++){
    console.log(fn[i]());
}

    根据上述结论，我们可以得出最后的输出就是10~19。因为最后的i值为10，然后由于i一直在内存当中，后面每次执行fn[i]，i就会加1。

    function a(){
    var arr = new Array();
    for (var i=0; i < 10; i++){
        arr[i] = function(){
            return i;
        }();                        //让函数立即执行
    }
    return arr;
}
var fn = a();
for (var i=0; i < fn.length; i++){
    console.log(fn[i]);
}

    要让这个函数输出符合我们预期的结果，有几种简单的改法。可以让函数a里面的匿名函数立即执行，这样就不会等到函数全部定义完然后i值为10才执行。当然，输出的时候就不需要再执行了。

    function a(){
    var arr = new Array();
    for (var i=0; i < 10; i++){
        arr[i] = function(i){
            return function(){
                return i;
            }
        }(i);
    }
    return arr;
}
var fn = a();
for (var i=0; i < fn.length; i++){
    console.log(fn[i]());
}

    # 六、总结

    看了很多网上对闭包的解释，参考了各路大神对闭包的理解。说实话，最后还是不知道自己到底是理解了闭包还是没有真正理解闭包。也许实际上只是明白了几种闭包的用法吧。因为对各种概念直接去理解实在是有些抽象，最后只能结合例子一步步理解，这样相对容易理解一点，但是也无形之中把自己对闭包的理解局限在这些例子当中。以后有机会还是要更好的去理解闭包。
