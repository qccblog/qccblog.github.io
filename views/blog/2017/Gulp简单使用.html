    Gulp简单使用

    qiucong 2017/6/25 gulp

    # 安装

    首先需要有一个node的环境，然后再全局安装gulp，有需要请自行加上sudo，网速不好的话，npm命令后面可以加上--registry=https://registry.npm.taobao.org或者直接使用npm config set registry https://registry.npm.taobao.org更换npm下载软件的源。

    npm install gulp -g

    新建一个文件夹作为项目，使用npm init初始化配置package.json文件。这个时候我们直接使用gulp命令会提示我们
    Local gulp没有找到，需要使用npm install gulp命令安装。
    npm install gulp --save-dev
    我们已经可以使用gulp命令，在根目录新建一个名为gulpfile.js的文件，这个就是用来配置gulp各种设置的地方。

    // 引入 gulp
var gulp = require('gulp');

gulp.task('default', function(){
    console.log('我是gulp');
});

    # API

    # gulp.src(globs[, options])

    在Gulp中，使用的是Nodejs中的stream(流)，首先获取到需要的stream，然后可以通过stream的pipe()方法把流导入到你想要的地方，比如Gulp的插件中，经过插件处理后的流又可以继续导入到其他插件中，当然也可以把流写入到文件中。
    globs参数是文件匹配模式(类似正则表达式)，用来匹配文件路径(包括文件名)，当然这里也可以直接指定某个具体的文件路径。当有多个匹配模式时，该参数可以为一个数组。
    options为可选参数。通常情况下我们不需要用到。

    # gulp.dest(path[, options])

    主要使用来写文件，path是文件输出的目录，如果不存在会自动创建。

    # gulp.task(name[, deps], fn)

    定义一个使用 Orchestrator 实现的任务（task）。
    name就是你定义的这个任务的名称，deps是一个需要在当前任务之前执行的任务列表，fn是定义任务执行哪些操作。

    # gulp.watch(glob[, opts], tasks)

    监视文件，并且可以在文件发生改动时候做一些事情。
    glob匹配模式用来指定具体监控哪些文件的变动；opts 为一个可选的配置对象，通常不需要用到；tasks是需要在文件变动后执行的一个或者多个通过 gulp.task() 创建的 task 的名字。

    # gulp.watch(glob[, opts, cb])

    和上一个不同之处在于cb是文件每次变动需要执行的 callback。

    # 常用插件

    gulp的强大之处在于他可以简单的使用插件简化任务。有一些常用的插件可以帮助我们快速方便的使用。

    1. 自动加载插件(gulp-load-plugins)
      要使用gulp的插件，首先得用require来把插件加载进来，如果我们要使用的插件非常多，那我们的gulpfile.js文件开头就会是一堆require，使用该插件可以按需加载已经正确安装并在package.json文件里面的gulp插件。
    2. js代码检查(gulp-jshint)
    3. less文件编译(gulp-less)
    4. 文件合并(gulp-concat)
    5. js文件压缩(gulp-uglify)
    6. css文件压缩(gulp-minify-css)
    7. html文件压缩(gulp-minify-html)
    8. 图片压缩(gulp-imagemin)
    9. 自动刷新(gulp-livereload)
      该插件最好配合谷歌浏览器来使用，且要安装livereload chrome extension扩展插件
    10. 重命名(gulp-rename)

    # 使用

    var gulp = require('gulp');

// 引入组件
var jshint = require('gulp-jshint');
var less = require('gulp-less');
var concat = require('gulp-concat');
var uglify = require('gulp-uglify');
var rename = require('gulp-rename');

// 检查脚本
gulp.task('lint', function() {
    gulp.src('./public/js/*.js')
        .pipe(jshint())
        .pipe(jshint.reporter('default'));
});

// 编译less
gulp.task('less', function() {
    gulp.src('./public/less/*.less')
        .pipe(less())
        .pipe(gulp.dest('./public/css'));
});

// 合并，压缩文件
gulp.task('scripts', function() {
    gulp.src('./public/js/*.js')
        .pipe(concat('all.js'))
        .pipe(gulp.dest('./public/build'))
        .pipe(rename('all.min.js'))
        .pipe(uglify())
        .pipe(gulp.dest('./public/build'));
});

// 默认任务
gulp.task('default', function(){
    // 监听文件变化
    gulp.watch('./public/js/*.js', ['lint', 'less', 'scripts']);
});

gulp.watch('./public/js/*.js', function(event){
    console.log(event.type); //变化类型 added为新增,deleted为删除，changed为改变
    console.log(event.path); //变化的文件的路径
});
