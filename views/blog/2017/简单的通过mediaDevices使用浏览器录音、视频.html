    简单的通过mediaDevices使用浏览器录音、视频

    qiucong 2017/8/19 htmljs

    想着能不能实现语音的功能，发现HTML5是有相关的API的。

    # 开始录音

    我们可以使用MediaDevices.getUserMedia (opens new window)方法来提示用户允许使用一个视频和/或一个音频输入设备。在旧版本浏览器中使用Navigator.getUserMedia() (opens new window)方法，但是该方法正在被弃用。

    navigator.mediaDevices.getUserMedia({audio: true, video: false})
.then(function(mediaStream) {
  //处理获得的媒体内容的流
})
.catch(function(error) {
  //捕获异常，比如用户不允许开启或者硬件错误之类的
});

    # 关闭录音

    发现开启了录音功能之后，不刷新的话会一直开着。看了半天的getUserMedia()的文档，也没有发现关闭的接口。网上找的基本上都是不能用的，后面看到MediaStreamTrack比较像某个网上说可行的方法，发现果然有stop方法，只能说这种实验室性质的Api变化还是很快的，参考文档 (opens new window)。

    stream.getAudioTracks()[0].stop();
    1

    # 创建媒体对象的URL

    媒体内容的流是不能直接使用，需要经过处理后才能转换成需要的音频或者视频URL。

    # 直接通过MediaStream

    使用一个MediaStream对象作为createObjectURL的输入正在被弃用，在最新浏览器上应该使用HTMLMediaElement.srcObject()替代

    var audio = document.querySelector('audio');
if ("srcObject" in audio) {
  audio.srcObject = stream;
} else {
  audio.src = window.URL.createObjectURL(stream);
}

    # 通过Blob创建

    var audio = document.querySelector('audio');
var blob = new Blob([data], { type: 'audio/wav' });
audio.src = window.URL.createObjectURL(blob);

    # 传输获得的一段音频

    我在使用的时候，如果直接把媒体内容的流传到服务器再发送给用户，发现并不能正确的解析。所以我找了把媒体内容的流处理之后形成Blob对象。一个Blob对象表示一个不可变的，原始数据的类似文件对象。还需要对音频进行处理。参考http://web.jobbole.com/84706/。

    HTML5有很多强大的API，为web应用提供了非常丰富的接口，期待对各种特性的支持越来越好。
