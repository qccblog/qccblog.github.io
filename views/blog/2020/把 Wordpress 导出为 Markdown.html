    把 Wordpress 导出为 Markdown

    qiucong 2020/3/1 安装配置

    Wordpress 使用起来非常方便，但是现在用起来觉得有些臃肿了，扩展起来相对而言也比较费力，就想着把 Wordpress 里面的内容导出为 markdown 以便于迁移到其他工具中。

    这个春节在家宅的时间实在是太长，正好可以动手把一直要迁移的 Wordpress 给搞定了。Wordpress 在以前可以说是全世界范围内非常流行的博客了，更新迭代周期快，功能强大，有相当多的主题和插件可用。但是不好的地方就是前端渲染比较慢，再加上后台引用了一些被墙的 cdn 和 avatar 库，如果不好好配置一番，在后台写文章体验会相当不好。

    这次把博客从 Wordpress 迁移到 VuePress 上，除了 VuePress 怎么选主题外就是要看怎么把以前在 Wordpress 上的博客给迁移到 VuePress 上。由于 VuePress 用的是 markdown 格式的文章，所以只要把 Wordpress 里面的文章导出成 markdown 格式就行了。

    # 借助 Hexo

    Hexo 是由 Node.js 驱动的一款快速、简单且功能强大的博客框架。可以快速将 Markdown 格式的文档生成静态页面。是越来越笨重的 Wordpress 和麻烦的空间管理的替代方案。

    # 新建一个hexo工程
mkdir hexo
cd hexo

# 初始化hexo
npx hexo init

# 安装 hexo-migrator-wordpress 插件
npm install hexo-migrator-wordpress --save

# 登录 Wordpress 后台，在“工具”-“导出”中导出所有记录。
# source 是 Wordpress 导出文件的存放目录。
npx hexo migrate wordpress <source>

    迁移完成之后可以在 source/_posts 文件夹下里面看到你的文章了，不过要注意的是如果是中文标题会被转码，不过没关系，点进去文章里面的 title 还是中文的。

    # wordpress-to-markdown

    可以不用借助 hexo，直接使用 nodejs 来进行操作。

    wordpress-to-markdown (opens new window) 使用 xml2js (opens new window) 解析 XML，然后使用 to-markdown (opens new window) 将 HTML 发布内容转换为 Markdown。还会自动下载博客中的图片存储到

    git clone https://github.com/ytechie/wordpress-to-markdown.git
cd wordpress-to-markdown/
npm install xml2js to-markdown

# 把从Wordpress中导出的文件重命为export.xml放到文件夹中
node convert.js
