    少年，FRP 内网穿透了解一下？

    qiucong 2020/3/22 安装配置

    正常情况下直接从公网中访问不在同一个局域网内的设备是比较麻烦的，特别是现在运营商因为 IP 地址短缺也很难分配公网 IP 地址。如果想要远程访问这些设备一般来说要通过一些转发或者 P2P 组网软件的帮助。

    # 常用方式

    • 使用 TeamViewer 进行远程。可以直接远程桌面，不过正式版经常提醒购买或者有时长限制（ps：限制越来越多了）。
    • 使用向日葵远程，免费版本网络感人，体验不佳，用起来一卡一卡的。
    • 使用 ngrok 或 frp 这类工具进行内网穿透，可以自己搭建需要公网 ip，也可以使用服务商提供的免费版或付费版服务。

    # FRP

    FRP (opens new window) 全名：Fast Reverse Proxy。FRP 是一个使用 Go 语言开发的高性能的反向代理应用，简单配置后就可以进行内网穿透，对外网提供服务。FRP 支持 TCP、UDP、HTTP、HTTPS 等协议类型，并且支持 Web 服务根据域名进行路由转发。FRP 由一个 server 端（一般是有公网 IP 的服务器），一个 client 端（一般是在内网需要被访问的机器）组成。

    # 下载

    FRP 不需要进行安装，只需要从 GitHub 上下载之后解压配置之后就可以使用。下载地址为 https://github.com/fatedier/frp/releases (opens new window)，点进去会发现有很多不同版本不同系统可以进行下载，当前最新版本是 v0.32.0，一般选择最新版本下载即可。

    对于 Windows 操作系统只要选择对应 Windows 版本即可，对于 Linux 可以先通过 cat /proc/version 或 arch 查看一下处理器架构，根据架构下载不同版本的 FRP。

    下面以 amd64 版本为例：

    # 下载
wget https://github.com/fatedier/frp/releases/download/v0.32.0/frp_0.32.0_linux_amd64.tar.gz
# 解压
tar -zxvf frp_0.32.0_linux_amd64.tar.gz
# 文件夹改个名
cp -r frp_0.32.0_linux_amd64 frp

cd frp

    # server 端

    启动 server 端的过程很简单，只要执行 frps 即可。

    # 直接启动
./frps

# 输出以下内容就启动成功了
[I] [service.go:157] frps tcp listen on 0.0.0.0:7000
[I] [root.go:209] start frps success

    默认配置往往不能满足我们的全部需求，这时候就要通过改配置文件来修改配置以满足需求。

    通过 frps_full.ini 可以看到有很多配置，稍微列一下常用的一些配置：

    • bind_port：表示用 client 端和 server 端连接的端口，这个端口号我们之后在配置 client 端的时候要用到。
    • dashboard_port：是 server 端仪表板的端口，若使用 7500 端口，在配置完成服务启动后可以通过浏览器访问 x.x.x.x: 7500 （其中 x.x.x.x 为 server 端的 IP）查看 frp 服务运行信息。
    • token：是用于 client 端和 server 端连接的口令，为了防止恶意连接最好配置。
    • dashboard_user、dashboard_pwd：表示打开仪表板页面登录的用户名和密码，自行设置即可。
    • vhost_http_port、vhost_https_port：用于反向代理 web 服务时使用。

    # client 端

    启动 client 端需要稍微修改一下 frpc.ini 文件来连接 server 端。

    # 编辑配置文件
vi frpc.ini

# 默认配置
[common]
server_addr = 127.0.0.1
server_port = 7000

[ssh]
type = tcp
local_ip = 127.0.0.1
local_port = 22
remote_port = 6000

    # 使用实战

    结合几个常用场景给出实际使用的例子，这边假定 server 端有一个公网 ip： 114.114.114.114。

    # ssh 访问服务器

    在之前的 frpc.ini 配置文件中有关于 ssh 的一段配置，两边都启动成功之后就可以连接到 client 端的服务器了，使用上和直接连接到 client 没有区别。

    ssh -p 6000 root@114.114.114.114
    1

    通过 ssh 连接的时候通过-p 指定端口为 client 中设置的 remote_port，用户和密码是 client 端的，ip 地址是 server 端的。

    # 通过自定义域名访问部署于内网的 Web 服务

    有时需要在公有网络通过域名访问我们在本地环境搭建的 Web 服务，但是由于本地环境机器并没有公网 IP，无法将域名直接解析到本地的机器。通过 FRP 就可以很容易实现这一功能，这里以 HTTP 服务为例：首先修改 FRP 服务端配置文件，通过 vhost_http_port 参数来设置 HTTP 访问端口，这里将 HTTP 访问端口设为 8080。

    # 修改server配置
vi frps.ini

[common]
bind_port = 7000
dashboard_port = 7500
vhost_http_port = 8080

# 启动 FRP 服务端
./frps -c ./frps.ini

# 输出日志

[web]

    # 通过 frpc 所在机器访问外网

    HTTP 代理插件，用于将内网机器作为 HTTP 代理暴露给其他服务，可以通过此代理访问到此内网机器能够访问到的其他服务。

    frps 的部署步骤同上。

    启动 frpc，启用 http_proxy 或 socks5 插件(plugin 换为 socks5，然后修改参数名称即可)， 配置如下：

    # frpc.ini
[common]
server_addr = x.x.x.x
server_port = 7000

[http_proxy]
type = tcp
remote_port = 6000
plugin = http_proxy
plugin_http_user = abc
plugin_http_passwd = abc

    # 扩展：将服务端 frps 添加为 service

    通过在终端直接启动 frps 的方式，会占用终端，并且停止和重启等管理操作也不方便。为了方便管理 frps 的启动、停止、重启和开机自启，我们需要将 frps 添加为系统 service，通过 systemd 守护程序来管理。

    在 frp 目录里的 systemd 目录下，已经为我们准备好了添加 service 的模板：

    • frpc.service —— client 端使用此模板
    • frpc@.service
    • frps.service —— server 端使用此模板
    • frps@.service
    # 进入目录frp/systemd
cd frp/systemd

# 编辑frps.service
vi frps.service

# 服务运行配置项ExecStart=/usr/bin/frps -c /etc/frp/frps.ini和直接启动类似，需要修改为合适的地址

# 复制frps可执行文件到/usr/bin目录
cp frp/frps /usr/bin

# 复制frps.ini到/etc/frp目录
cp frp/frps.ini /etc/frp

# 复制frps.service到/etc/systemd/system目录
cp frp/systemd/frps.service /etc/systemd/system


# 启动frps
systemctl start frps

# 停止frps
systemctl stop frps

# 重启frps
systemctl restart frps

# 查看frps状态
systemctl status frps

# 开机启动frps
systemctl enable frps

    # 通过 docker 部署

    frps 的部署可以说已经非常简便了，不过基于各种原因还是可以通过 docker 进行快速部署的，本地一份配置文件加上开放的端口即可。

    docker run -p 7000:7000 -v /home/frp/frps.ini:/etc/frps.ini -d frankqcc/frps

# -net设置为host可以不用配置端口映射
# 测试的时候可以方便不用不停切换端口，正式环境不建议这么使用
docker run --rm -d --net=host --name frps -v /home/fr/frp/frps.ini:/etc/frps.ini frankqcc/frps

    如果需要修改配置文件就要把本地的配置文件映射到 docker 中，使用上和直接部署没有任何区别。client 端同理也可以进行 docker 部署，不过个人觉得实际使用过程中直接使用 frpc 镜像的场景不多。
