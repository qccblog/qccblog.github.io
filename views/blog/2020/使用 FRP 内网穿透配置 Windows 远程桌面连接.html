    使用 FRP 内网穿透配置 Windows 远程桌面连接

    qiucong 2020/12/8 安装配置

    对于想要访问内网或者没有公网 IP 的 Windows 用户来说这并不是一件容易的事情，使用 TeamViewer 或者向日葵需要购买之后访问速度才能流畅的使用，而且还得考虑安全问题。因此很多人考虑自行搭建一个代理来进行内网穿透。这边主要介绍怎么使用 FRP 来访问 Windows 远程桌面，如果只是想要知道怎么配置看第一段就行了，如果考虑安全或者流量比较大可以继续看到最后。

    首先从 Github 上下载 frp (opens new window)，如果之前没有接触过 FRP 可以参考上一篇文章 使用 FRP 进行内网穿透 或者去 官方文档 (opens new window) 了解相关内容。

    # 配置服务端

    修改 frps.ini 文件

    [common]
#Bind all local network interfaces
bind_addr = 0.0.0.0 
#Expose local port for client to connect with
bind_port = 7001
#Connection password to validate client
token = xxx

# Dashboard settings

    # 配置 frp

    修改 frpc.ini 文件

    [common]
server_addr = xx.xxx.xx.xx
server_port = 7001
token = xxx  # 密码，和服务器一样
 
[RDP] 
type = tcp
local_ip = 0.0.0.0
# RDP default port
# Default port for RDP protocol is 3389
# local_port is to map local_ip:3389 to remote_ip:3399
# 远程桌面的端口号
local_port = 3389
# 远程桌面服务暴露在服务端的端口号
remote_port = 3399

    # 开启远程访问

    设置 =》系统 =》远程桌面 =》启用远程桌面，这边可以配置远程访问的账号密码，默认会把当前的账户密码用来远程访问。

    # 外网访问

    各个平台下输入 ip: 3399 远程控制和账户名、密码即可。

    如果是用 Windows 进行远程控制，那么直接使用系统自带的远程桌面连接就可以了。

    如果想在其他平台或者在移动端使用，可以安装名为 Microsoft remote desktop 的 app，和 Windows 下远程访问类似了。

    只是想要简单配置一个可以访问远程桌面就已经完全没问题了。

    # STCP

    上述的方案是直接把内网的电脑暴露在公网环境中，出于安全考虑，可能并不希望这种连接方式。使用 stcp（secret tcp）类型的代理可以避免让任何人都能访问到要穿透的服务，但是访问者也需要运行另外一个 frpc 客户端。

    服务端：配置仍然同默认配置一致，直接运行即可。

    被访问客户端 1：配置需要将 type 改为 stcp，并且增加一个 sk 字段。这里不需要远端端口（remote_port），因为服务是不暴露在公网的。

    # frpc.ini
[common]
# 你的frp服务器的公网ip
server_addr = x.x.x.x
# 你的frp服务器的默认端口
server_port = 7000

[rdp]
type = stcp
# 只有 sk 一致的用户才能访问到此服务
sk = abcdefg
local_ip = 127.0.0.1
# 远程桌面的本地端口号
local_port = 3389

    访问客户端 2：访问者也需要通过 frpc 客户端进行连接才能访问。

    # frpc.ini
[common]
# 你的frp服务器的公网ip
server_addr = x.x.x.x
# 你的frp服务器的默认端口
server_port = 7000

[rdp_visitor]
type = stcp
# stcp 的访问者
role = visitor
# 要访问的 stcp 代理的名字
server_name = rdp
# 只有 sk 一致的用户才能访问到此服务
sk = abcdefg
# 绑定本地端口用于访问 远程桌面 服务
bind_addr = 127.0.0.1
bind_port = 6000

    # 点对点内网穿透（xtcp 模式）

    frp 提供了一种新的代理类型 xtcp 用于应对在希望传输大量数据且流量不经过服务器的场景。

    使用方式同 stcp 类似，需要在两边客户端都部署上 frpc 用于建立直接的连接。

    目前处于开发的初级阶段，并不能穿透所有类型的 NAT 设备，所以穿透成功率较低。穿透失败时可以尝试 stcp 的方式。

    frps 除正常配置外需要额外配置一个 udp 端口用于支持该类型的客户端:

    bind_udp_port = 7001
    1

    启动 frpc，转发内网的 ssh 服务，配置如下，不需要指定远程端口:

    # frpc.ini
[common]
server_addr = x.x.x.x
server_port = 7000

[p2p_ssh]
type = xtcp
# 只有 sk 一致的用户才能访问到此服务
sk = abcdefg
local_ip = 127.0.0.1
local_port = 22
