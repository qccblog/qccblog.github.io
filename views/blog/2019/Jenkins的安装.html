    Jenkins的安装

    qiucong 2019/1/12 jenkins安装配置

    # 安装

    官网 (opens new window)下载地址。 简便安装：

    1. 下载jenkins.war (opens new window)，安装说明 (opens new window)。对于有tomcat的小伙伴，直接扔tomcat的webapps里面启动tomcat，访问tomcat工程地址/jenkins
    2. docker安装
      1. 官方镜像jenkins/jenkins (opens new window)，通过sudo docker run -p 8080:8080 -p 50000:50000 -v /opt/data/jenkins:/var/jenkins_home --name jenkins jenkins/jenkins 运行。/opt/data/jenkins是定义在宿主上存放数据的地址，Jenkins是以文件夹的形式存放数据的。8080为访问Jenkins Server页面端口，50000为创建的Jenkins Slave与Master建立连接进行通信的默认端口。
      2. 官方推荐安装的镜像是jenkinsci/blueocean (opens new window)，里面包含Blue Ocean插件。查看安装运行说明 (opens new window)
      3. 个人推荐使用frankqcc/jenkins:docker (opens new window)，支持支持docker功能，内置allure。直接通过sudo docker run -p 50000:50000 -p 8060:8080 --name jenkins -d -v /var/run/docker.sock:/var/run/docker.sock --restart=always -v /opt/data/jenkins:/var/jenkins_home frankqcc/jenkins:docker即可使用。

    # 新增节点

    可以通过配置Slave节点来提高Jenkins执行的效率。

    # jnlp-slave (opens new window)

    # docker 运行jenkins-slave示例，使用jnlp方式进行连接，即是agent主动连接master
docker run jenkins/jnlp-slave -url http://jenkins-server:port -workDir=/home/jenkins/agent <secret> <agent name>
    1
    2

    首先在Jenkins中新增一个系统管理-》节点管理-》新建节点。 保存之后会看到新建的这个节点有个红色的x，这是因为还没有这个节点，无法建立连接，接下来就是新建这个节点。

    docker run --name jenkins-slave02 -d --restart=unless-stopped jenkins/jnlp-slave -url  http://192.168.1.232:8060 -workDir=/home/jenkins/agent02 secret agent02
    1

    其中url参数就是现有Jenkins服务器的地址，workDir就是刚刚配置时候设置的，secret就是下图的-secret后面的值。这些配置信息都可以点这个节点然后查看。

    使用以下镜像直接支持在slave中执行docker。

    docker run --name jenkins-slave02 -d -v /opt/slave/agent02:/home/jenkins/agent -v /var/run/docker.sock:/var/run/docker.sock --restart=unless-stopped frankqcc/jnlp-slave -url http://192.168.1.232:8060 -workDir=/home/jenkins/agent secret agent02
    1

    需要注意的是这种方式运行的节点上执行docker都是宿主机上的行为。比如-v /home/abc:/home中的/home/abc就是运行docker宿主机上的路径。

    # 配置代理

    简单贴了一下官网给出的通过nginx转发到Jenkins。

    upstream jenkins {
  keepalive 32; # keepalive connections
  server 127.0.0.1:8080; # jenkins ip and port
}
 
server {
  listen          80;       # Listen on port 80 for IPv4 requests

  server_name     jenkins.example.com;

  #this is the jenkins web root directory (mentioned in the /etc/default/jenkins file)
  root            /var/run/jenkins/war/;

  access_log      /var/log/nginx/jenkins/access.log;
  error_log       /var/log/nginx/jenkins/error.log;
  ignore_invalid_headers off; #pass through headers from Jenkins which are considered invalid by Nginx server.

  location ~ "^/static/[0-9a-fA-F]{8}\/(.*)$" {
    #rewrite all static files into requests to the root
    #E.g /static/12345678/css/something.css will become /css/something.css
    rewrite "^/static/[0-9a-fA-F]{8}\/(.*)" /$1 last;
  }

  location /userContent {
    #have nginx handle all the static requests to the userContent folder files
    #note : This is the $JENKINS_HOME dir
	root /var/lib/jenkins/;
    if (!-f $request_filename){
      #this file does not exist, might be a directory or a /**view** url
      rewrite (.*) /$1 last;
	  break;
    }
	sendfile on;
  }

  location / {
      sendfile off;
      proxy_pass         http://jenkins;
      proxy_redirect     default;
      proxy_http_version 1.1;

      proxy_set_header   Host              $host;
      proxy_set_header   X-Real-IP         $remote_addr;
      proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
      proxy_set_header   X-Forwarded-Proto $scheme;
      proxy_max_temp_file_size 0;

      #this is the maximum upload size
      client_max_body_size       10m;
      client_body_buffer_size    128k;

      proxy_connect_timeout      90;
      proxy_send_timeout         90;
      proxy_read_timeout         90;
      proxy_buffering            off;
      proxy_request_buffering    off; # Required for HTTP CLI commands in Jenkins > 2.54
      proxy_set_header Connection ""; # Clear for keepalive
  }

}

    # 遇到的问题

    # Jenkenis报错：该jenkins实例似乎已离线

    验证密钥之后，过了好久提示可能是离线。可以新开一个http://localhost:8080/pluginManager/advanced页面，把最下面的【升级站点】中的链接改成http形式的http://updates.jenkins.io/update-center.json，建议改成https://mirrors.tuna.tsinghua.edu.cn/jenkins/updates/update-center.json的地址。重新访问即可，还有问题就需要重启一下看看。或者直接修改/opt/data/jenkins/hudson.model.UpdateCenter.xml里面对应的内容。

    # 下载插件速度慢

    由于update-center.json中很多地址并没有替换为镜像地址，所以下载插件时还是会直接访问 http://updates.jenkins-ci.org。

    # 配置Nginx代理update.jenkins-ci.org

    推荐的方案。

    # 第一步：将updates.jenkins.org映射到本地nginx
vi /etc/hosts

127.0.0.1 updates.jenkins-ci.org

# 第二步：将请求映射到镜像网站
vi /etc/nginx/conf.d/default.conf

location /download/plugins
{
    proxy_next_upstream http_502 http_504 error timeout invalid_header;
    proxy_set_header Host mirrors.tuna.tsinghua.edu.cn;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    rewrite /download/plugins(.*) /jenkins/plugins/$1 break;
    proxy_pass https://mirrors.tuna.tsinghua.edu.cn;
}

    # 直接修改文件

    这种方式适合不怎么使用插件的临时使用方案。

    # 进入更新配置位置
cd /opt/data/jenkins/updates

# 替换所有插件下载的url和连接测试url
sed -i 's/http:\/\/updates.jenkins-ci.org\/download/https:\/\/mirrors.tuna.tsinghua.edu.cn\/jenkins/g' default.json && sed -i 's/http:\/\/www.google.com/https:\/\/www.baidu.com/g' default.json

    # Permission denied报错

    需要给我们挂载的目录对应的权限sudo chown -R 1000 /opt/data/jenkins

    # Jenkins/密码正确却突然登不上去（一直显示密码错误）

    1. 修改工作目录（JENKINS_HOME指向的目录）下的config.xml的true为false
    <useSecurity>true</useSecurity>
    1
    1. 重启Tomcat（重启Jenkins）
    2. 进入首页> "系统管理" > "全局安全配置"
    3. 在安全域中选择“jenkins专有用户数据库”，并保存
    4. 进入“系统管理”>“管理用户”
    5. 从用户列表选择要登录的那个账号（以admin为例），进入设置，拉到下面，修改密码并保存
    6. 即可重新登录了。（但此时是跳过验证状态，把第一步改的false改为true，并重启Tomcat生效）
